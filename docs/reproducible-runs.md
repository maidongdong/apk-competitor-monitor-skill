# Reproducible Runs

Different agents can produce different APK reports when they silently use different state files, APK pairs, report templates, Web routes, dependency versions, or notify/publish settings. Treat the project config as the shared contract.

## Required Shared Inputs

- Use the same `project_config` file for every MCP run.
- Keep `source.app_url` identical. APK monitoring intentionally supports Wandoujia pages only.
- Keep `app.package` identical so source/API filtering uses the same package owner prefix.
- Keep `apk.state_file` and `apk.report_root` identical, or explicitly provide the same `apk_report_dir` for reusing an old report when no newer APK exists.
- Keep `apk.preview_limit` identical because it changes how many static UI previews enter the report.
- Keep `web.route_ids` identical for Web/admin comparisons.
- Keep `publish.url` and `notify.wecom_webhook_url` project-specific. Do not reuse another team's webhook.

## Validation Flow

Run the MCP tool `validate_project_config` before a scheduled monitor run. It returns:

- blocking config issues
- non-blocking reproducibility warnings
- config SHA256
- canonical config fingerprint
- direct override fingerprint

If two agents disagree on report contents or HTML, compare these fingerprints first. If they differ, the reports are not from the same run contract.

## Report Metadata

APK detail reports generated by `run_full_weekly_pipeline` write `run-metadata.json` into the report directory and embed the same object in `report-data.json` under `runMetadata`.

The HTML report shows this on the coverage page under "运行元数据":

- tool version
- project config fingerprint
- project config SHA256
- report template fingerprint
- direct override keys
- reproducibility flags

This does not make reverse engineering fully deterministic, because local jadx/apktool versions can still affect source mapping. It does make differences visible enough to debug quickly.

## 今日水印相机 Baseline

Use `apk-competitor-monitor/examples/today-watermark-camera.config.example.json` as the starting contract for 今日水印相机:

- app name: `今日水印相机`
- package: `com.xhey.xcamera`
- Wandoujia page: `https://www.wandoujia.com/apps/7912851`
- static preview limit: `50`

Copy it into your monitoring workspace, then fill only your own Web/admin route ids, publish URL, and WeCom webhook.
